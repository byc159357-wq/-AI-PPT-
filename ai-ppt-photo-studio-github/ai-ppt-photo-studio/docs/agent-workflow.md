# 多 Agent 工作流

## 总流程

```text
Input
  ↓
Research Agent
  ↓
Planning Agent
  ↓
Visual Agent
  ↓
Review Agent
  ↓
Output
```

---

## 1. Research Agent / 资料理解 Agent

### 输入

- 用户主题
- 文档内容
- 图片资料
- PDF 内容
- 参考风格

### 输出

- 关键词
- 核心信息
- 内容分类
- 可视化方向
- 资料摘要

### 示例

输入：

```text
制作一个关于三孔桥信息可视化设计的前期调研 PPT
```

输出：

```text
核心板块：
1. 桥体结构
2. 纹样图案
3. 民俗文化
```

---

## 2. Planning Agent / 内容策划 Agent

### 任务

- 生成 PPT 目录
- 规划页面数量
- 设计信息层级
- 判断每页适合的视觉表达方式

### 输出示例

```text
第 1 页：项目封面
第 2 页：研究背景
第 3 页：调研框架
第 4 页：桥体结构分析
第 5 页：结构信息可视化
```

---

## 3. Visual Agent / 视觉生成 Agent

### 任务

- 生成图片提示词
- 设计海报画面
- 生成图标建议
- 规划色彩和版式

### 输出示例

```text
中国古桥信息可视化海报，米白色宣纸背景，青灰色桥体线稿，局部朱红色标注，简洁学术风格
```

---

## 4. Review Agent / 设计评审 Agent

### 任务

检查最终结果是否存在问题：

- 信息是否清楚
- 画面是否拥挤
- 风格是否统一
- 是否有 AI 生成痕迹
- 是否符合用户主题

### 输出示例

```text
问题：页面元素过多，主标题不突出。
建议：减少装饰元素，将重点信息集中在页面左侧，并加强标题对比。
```
